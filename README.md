# HPC_Lab0_MatMul

# Использовалось

- Python
- NumPy
- Numba (CUDA)
- Google Cola

# Реализация
**CPU**

Реализация выполнена с использованием трёх вложенных циклов:

* внешний цикл по строкам

*	внутренний цикл по столбцам

*	цикл суммирования

Особенности:

*	последовательное выполнение

*	высокая вычислительная сложность O(N³)

*	низкая производительность на больших данных

**GPU**

Реализация выполнена с использованием CUDA через библиотеку Numba

# Распараллеливание

Каждый поток GPU вычисляет один элемент результирующей матрицы:

 - 1 поток → 1 элемент
- потоки работают независимо
- используется двумерная сетка потоков

# Результаты 
| Размер  | CPU (сек) | GPU (сек) | Ускорение |
| ------- | --------- | --------- | --------- |
| 100×100 | 0.7180    | 0.0012    | 609x      |
| 300×300 | 16.1076   | 0.0029    | 5464x     |
| 500×500 | 76.7064   | 0.0092    | 8333x     |
| 800×800 | 319.4392  | 0.0319    | 10023x    |

- При увеличении размера матриц время CPU резко возрастает (O(N³))
- GPU показывает значительно более стабильное время выполнения
- Ускорение увеличивается с ростом размера задачи

# Итог

Распараллелено вычисление эл-в результатирующей матрицы. каждый элемент вычисляется независимо. Это позволяет выполнять вычисления параллельно на большом количестве потоков GPU. 

GPU эффективно ускоряет задачи, в которых возможен высокий уровень параллелизма (перемножение матриц)
